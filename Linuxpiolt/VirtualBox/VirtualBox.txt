當您開始對Linux作業系統產生興趣時是不是不敢直接把它裝到自己的硬碟上面？當您在使用Linux作業系統碰到Linux不能完成的任務時，是不是很想要一個windows來幫您的忙？VirtualBox可以幫助您虛擬出另外一台電腦來。
首先對VirtualBox做一個簡單的介紹：
VirtualBox是類似VMWare，Virtual PC的虛擬機軟體，最初由德國的InnoTek軟體公司出品，後來該公司被Sun並購，而最近Sun公司又被Oracle並購，所以它現在是Oracle公司的軟體，它歷史複雜，但在功能方面也是越來越強，越來越多。這麼一款X86虛擬軟體，不僅可以安裝在三大主流系統Windows、Linux、Mac OSx上面，還支援OpenSolaris、Software Developer Kit（SDK）等平臺，使用該虛擬軟體可以安裝linux、windows、solaris、OpenBSD和FreeBSD等作業系統。它還是開源的，是一個發佈在GPL許可之下的自由軟體。VirtualBox還有Remote Desktop Protocol（RDP）、iSCSI及USB的支援，在虛擬機器下就可以使用USB2.0的硬體裝置了。
下面筆者將介紹VirtualBox在Windows xp和Ubuntu9.04下的安裝、配置和使用。

在虛擬機器中上網，通信（中）
先對虛擬機器的網路模式進行探討，這裡只探討兩種模式，分別為「NAT」和「Bridge」。筆者的實體機器為局域網連入，使用DHCP獲取IP地址。
NAT模式（網路地址轉換）：
打開虛擬機器的「設定值」介面，在左側選擇「網路」，可以看到VirtualBox提供了四個配接卡，啟用配接卡1，并把「Attached to」設定為「NAT」。這個模式可以透過實體系統NAT後存取外部網路，是由應用程式VirtualBox提供的NAT，這個模式下虛擬機器可以通過實體系統進行上網，前提是實體機器已經聯網，並且可以連接到外部網路，也就是說可以連入Internet。設定為這個值時，在虛擬機器中的IP地址為10.0.2.15。使用該模式設定簡單，只要實體機器能夠上網，那麼虛擬機器也能，但是也有一個缺點，虛擬機器的網路地址在外網是不可見的，它無法與實體機器進行通信。
Bridge模式（橋接模式）：
首先在「偏好設定」下面的「網路」中添置一塊虛擬網卡，添置成功後，在Windows的「控制面板」-「網路連結」中會多出一個連接.（圖16）
接下來要做的就是將這兩個連接橋接起來，操作方法為，選中兩個連接，使用右鍵單擊并選擇「橋接」，將會多出一個橋接網路連結。（圖17）
再到虛擬機器的網路設定值中，將「Attached to」設定為「Host-only Adapter」，這樣啟動虛擬機器後和實體機器感覺像是在同一個網路交換器上面。筆者的實體機器的IP為10.101.12.57，虛擬機器也使用DHCP獲取IP為10.101.12.136。如此設定，虛擬機器和實體機器之間的通信就簡單多了。當然要在這個模式下上網的話，虛擬機器的聯網方式就和實體機器一樣的。
探討了網路模式之後，根據需要進行設定，這是筆者使用NAT模式之後在Ubuntu啟動Firefox上網。（圖18）
使用Bridge模式後可以在實體機器和虛擬機器之間互PING，兩個機器之間是互通的。
再提一下另一種模式「Bridged Adapter」，這個模式就輕鬆了，用戶不需要手動去建立橋接，VirtualBox會自動橋接好，實體機器和虛擬機器之間同樣是可以通信的。

遠程控制xp作業系統（中）
在進行遠程控制之前需要說明的是網路模式使用「brige模式」，為的是讓虛擬機器和實體機器可以相互通信，設定方法和前面一樣，為了方便起見，打開「設定值」-「網路」，「Attached to」選為「Bridged Adapter」，「Name」選為「eth0」。
有兩種方法進行遠程控制，一種是利用xp自身的服務，即在xp下開啟「允許遠程控制」，另一種方法是在VirtualBox中設定，打開「設定值」-「遠端顯示」，勾選「啟用VRDP伺服器」，虛擬機器將會扮演遠端桌面協定（RDP）伺服器，允許從遠端利用標準的RDP客戶端軟體來連接與操作虛擬機器（如果它在運行中） 。
先使用第一種方式進行試驗，開啟虛擬機器，使用右鍵單擊「我的電腦」選擇「屬性」，在「遠程」選項卡中勾選「允許用戶遠程連接到此計算機」，然後利用ipconfig指令或者在網路管理中得到此虛擬機器的IP地址，如筆者的IP為10.101.12.143，在同一局域網內找一台電腦進行測試，遠程控制成功。（圖32）
第二種方法筆者是在同一台機子上進行測試的，因為可以利用rdesktop這個優秀的軟體，在ubuntu下安裝rdesktop非常簡單，打開「終端機」輸入以下指令即可：
sudo apt-get install rdesktop
完成之後用vrdp模式啟動虛擬機器，方法為，打開「終端機」，輸入以下指令：
VBoxManage startvm -type vrdp xp
提示：其實VirtualBox的操作都可以利用指令來完成，讀者不妨試試將上面的「vrdp」改成「gui」，發生了什麼？ vrdp即在後台運行，沒有操作界面，而gui就是圖形模式的啟動。
完成之後，會有啟動成功的提示。（圖33）
這時只需等待幾分鐘就可以利用rdesktop遠端控制虛擬機器了，繼續在終端機中輸入：
rdesktop-vrdp 10.101.12.143
如果有條件的話也可以到另外的電腦試試，這時rdesktop會開啟一個新的視窗（圖34），這樣就可以控制虛擬機器了。

VirtualBox指令簡介（中）
VirtualBox除了方便易用的圖形介面外，還有強大的指令管理工具，使用指令的好處可以省去啟動虛擬機器顯示圖形介面的資源，要是在VirtualBox中安裝了多個虛擬機器並同時使用圖形介面啟動，對資源消耗是非常大的，下面就簡單介紹幾個指令：
1.VBoxManage list vms|ostypes|hdds|dvds|runningvms
vms：列出當前安裝的所有虛擬機器的資訊。
ostypes：列出vbox支援的作業系統類型。
hdds：列出虛擬磁碟的資訊。
dvds：列出虛擬光碟的資訊。
runningvms：列出當前正在運行的虛擬機器。
2.VBoxHeadless -startvm "name"將「name」替換成虛擬機器的名稱，這條指令也是使用vrdp方式啟動的。
3.VBoxManage controlvm name pause|resume|poweroff
pause：暫停虛擬機器。
resume：重新開始虛擬機器。
poweroff：關閉虛擬機器。
4.VBoxManage clonevdi name1 name2
快速克隆虛擬機器，安裝一個系統需要花很長的一段時間，比如之前已經安裝了一台xp虛擬機器，想再裝一台怎麼辦呢？很多人想到了複製一個 VDI檔案，但是複製過後的VDI檔案在同一台VirtualBox上是不能使用的，因為同一台VirtualBox中的VDI檔有重複的UUID，使用這個命令就可以復制另外一個虛擬機器出來。（圖35）
完成之後建立虛擬機器使用該VDI檔就可以了！

使用快照功能（中）

使用Portable VirtualBox（中）
Portable VirtualBox又是幹什麼的呢？由於VirtulaBox是開源的，所以任何人都可以修改源代碼並且重新發佈，有人就寫了這個Portable VirtualBox，它能夠把VirtualBox和Linux放在USB手指內，插入Windows系統即可啟動Linux，這樣您走到哪裡都能用上自己設定好了的Linux作業系統了。
Portable VirtualBox可以至如下網址下載：http://www.german-winlite.de/wbb/index.php?page=DownloadDBView&katID=3
下載好後解壓到USB手指，打開Portable VirtualBox，可以看到使用方法和安裝都與VirtualBox沒有什麽區別，這裡要提醒一下讀者，如果USB手指容量夠大，可以安裝Ubuntu，Fedora等作業系統，沒有足夠的容量也可以先試試安裝Gentoo，Archlinux基礎系統。筆者在這裡安裝了一個Archlinux，因為USB手指容量不夠大。
因為之前已經介紹過Import，Export虛擬機器，用戶也可以把已有的Linux作業系統Import到Portable VirtualBox中，這樣就不用重新安裝一遍了。

在VirtualBox開啟3D加速（中）
最新版本的VirtualBox可以開啟OpenGL 3D加速功能，支援Windows，Linux及Solaris作業系統，還要有如下幾個條件：
1.它僅適用於部分Windows，Linux及Solaris作業系統。
2.支援32位元版本的xp和vista，Direct3D目前還不支援。
3.Linux作業系統必須有內核2.6.27以上及X.org要1.5以上。
4.虛擬機器上必須安裝好VBoxGuestAdditions（客端額外功能）。
這麼看來之前安裝的兩個虛擬機器都滿足這些要求，分別測試一下：
首先測試的是xp虛擬機器，在「設定值」-「一般」-「基本」裡勾選「Enable 3D Acceleration」，並且把繪圖記憶體大小改得高些，以便能夠運行3D程式，然後開啟虛擬機器，這裡筆者安裝了一個OpenGL遊戲來進行測試。（圖43）

將實體機器轉換到VirtualBox（中）
這部份的內容將實現把實體XP系統導入到Ubuntu的VirtualBox中。
VMware公司提供了一個將物理系統轉換到虛擬機器的軟體，可以利用這個強大的軟體幫助實現轉換功能，先到http://www.vmware.com/products/converter/下載好軟體并安裝，筆者下載了一個3.0的版本，完成之後從桌面雙擊VMware Converter圖示打開。（圖46）
點擊「Import Machine」或者使用快速鍵「Ctrl+M」，出現「Welcome to the VMware Converter Import Wizard」對話框，點擊「下一步」。
出現選擇來源介紹對話框，繼續「下一步」，然後選擇要轉換的來源類型，選擇「Physical computer」，點擊「下一步」。（圖47）
問你要轉換的是那台機器，可以選擇遠程的機器，輸入IP登入后即可轉換，筆者選擇第二個「This local machine」，點擊「下一步」。（圖48）
接下來選擇要轉換的磁碟，實體機器的硬碟資訊都已經列出，可以調整轉換成虛擬硬碟的大小，一般選中「Ignore page file and hibernation file」，意為「忽略頁面檔案和休眠檔案」，點擊「下一步」。（圖49）
選擇虛擬機器類型為「VMware standalone virtual machine」，下一步。
輸入虛擬機器的名稱和位置，點擊「下一步」。（圖50）
接下來的「VM Options」和「Networks」保持預設值即可，當然根據需要可以做適當修改。
最後點擊「完成」等待轉換完畢，完成之後目標位置上會多出幾個vmdk檔。（圖51）
在Ubuntu下打開VirtualBox，VirtualBox是支援vmdk檔的，只要將剛才轉換過的vmdk檔作為虛擬硬碟就可以啟動了。（圖52）
這樣就實現了實體機器轉換成了VirtualBox虛擬機器。

VirtualBox使用體會（中）
這款軟體推出以來，受到了很多人的青睞，因為它是開放源代碼而且免費的，功能不弱於那些收費的虛擬機軟體，相信在開源力量的推動之下，這款軟體會越來越強大，而目前的功能對於用戶體驗系統已經足夠了。在使用過程中也可以發現使用該軟體並不是非常困難，但是想要用精就需要花一點功夫，比如掌握了vbox的指令系統就可以通過啟動腳本讓VirtualBox隨實體機器的啟動而啟動，還可以安排計劃任務。可以利用虛擬機器安裝Linux伺服器，伺服器的配置很麻煩，而且主要在指令行下面操作，利用VirtualBox可以一邊查看資料一邊進行伺服器的操作，等熟練掌握了伺服器的配置方法再實施到實體機器中，這對學習是非常有幫助的，如果把虛擬機器和實體機器設定在同一網段，還可以對伺服器進行測試，比如ssh服務連接正不正常，ftp伺服器是否運作等等。
